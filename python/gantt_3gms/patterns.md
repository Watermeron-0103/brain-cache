# スケジュールパターン解説【1日版と日単位版】

## 概要
このレポジトリでは、ガントチャートを作成するために、用途に応じて兩種類のスケジュールデータを扱います。  
一つは1日内で細かなタイムテーブルを記述する **パターンA**、もう一つは複数日にまたがるスケジュールを記述する **パターンB** です。  
用途やデータの形式が異なるため、CSVファイルの列構成とスクリプトがそれぞれ用意されています。

## パターンA（1日集中タイムテーブル）
- 1日内で複数のタスクの開始時刻と終了時刻を細かく記述する形式です。
- CSVファイルは以下の列を持ちます：

| 列名 | 説明 |
|---|---|
| Task | タスク名 |
| StartDate | 日付（YYYY-MM-DD、全タスク共通） |
| StartTime | 開始時刻（HH:MM） |
| EndTime | 終了時刻（HH:MM） |
| Owner | 担当者や参加者 |
| Notes | 補足説明（任意） |

- `StartDate` は1日集中の場合は同じ日付を入れます。  
- `StartTime` と `EndTime` で時間帯を指定し、`EndTime` は `StartTime` より後に設定します。
- Pythonの例では `single_day_from_csv.py` を使用して、時刻軸を横にしたガントチャートを生成します。
- CSV例（コードブロック）：

    Task,StartDate,StartTime,EndTime,Owner,Notes
    キックオフ,2025-10-06,09:00,09:30,井上,目的共有
    デモ(操作A),2025-10-06,09:30,10:30,赤坂,一連操作
    デモ(操作B),2025-10-06,10:40,11:40,山口,照明条件確認
    休憩,2025-10-06,11:40,12:00,全員,
    個別練習①,2025-10-06,13:00,14:30,小倉・今野,保存・エクスポート
    Q&A＋小テスト,2025-10-06,14:40,15:30,全員,チェックリスト

## パターンB（複数日計画: 日単位）
- 複数日にわたるタスクを日単位で管理する形式です。
- CSVファイルは以下の列を持ちます：

| 列名 | 説明 |
|---|---|
| Task | タスク名 |
| Start | 開始日（YYYY-MM-DD） |
| DurationDays | 期間（日数、0.5= 半日など） |
| Owner | 担当者や参加者 |
| Notes | 補足説明（任意） |

- `Start` はタスクの開始日です。 `DurationDays` が1.0なら1日、0.5なら半日など、タスクの長さを日数で表します。
- Pythonの例では `gantt_chart.py` を使用して、日付軸のガントチャートを生成します。
- CSV例（コードブロック）：

    Task,Start,DurationDays,Owner,Notes
    キックオフ,2025-10-03,0.5,井上,目的共有
    デモ測定(グループA),2025-10-04,1.0,赤坂・山口,一連の操作確認
    デモ測定(グループB),2025-10-05,1.0,小倉・今野,至室対応者含む
    個別操作練習,2025-10-06,1.0,全員,設定変更〜保存確認
    フォローQ&A,2025-10-07,0.5,全員,不明点吸い上げ
    確認テスト,2025-10-08,0.5,各自/確認:井上,合格で経練完了

## パターンAとパターンBの違い
以下の表は両パターンの主な違いをまとめたものです。

| 特徴 | パターンA（1日版） | パターンB（日単位版） |
|---|---|---|
| 期間の単位 | 時間（HH:MM） | 日数 |
| 必須列 | StartDate, StartTime, EndTime | Start, DurationDays |
| 具体的な用途 | 一日集中講義やタイムテーブルの詳細 | 数日間の研修全体や進捗管理 |
| スクリプト例 | single_day_from_csv.py | gantt_chart.py |

上記の形式に従ってCSVファイルを用意し、それぞれのPythonスクリプトを実行することで、目的に合ったガントチャートを作成できます。  
用途に応じて適切なパターンを選び、スケジュールの可視化に活用してください。
